Нужно доработать auth-сервис так, чтобы у пользователя была привязка активной сессии к конкретному устройству.

## Что сейчас происходит

Пользователь логинится на одном устройстве и получает пару токенов: access и refresh.

Если он логинится на другом устройстве, сервис выдает еще одну пару токенов.

В результате у одного пользователя может быть несколько активных сессий одновременно, и они должны храниться отдельно.

## Что нужно реализовать

При логине создавать сессию, привязанную к конкретному устройству.

Добавить ручку GET /security/devices, которая возвращает список активных устройств пользователя.

Все GET-операции должны идти через qweryRepository, который нужно реализовать.

Доступ к базе данных должен быть только через слой repository.

Для CRUD-операций использовать слой service.

### Требования к данным сессии

В базе можно использовать такую структуру:

user_id — id пользователя.

device_id — id устройства.

iat — время создания сессии.

device_name — название устройства, например chrome 121.

ip — IP-адрес клиента.

exp — время истечения сессии.

## Пример записи:

json
{
"user_id": 1,
"device_id": 1,
"iat": "2000-02-27T15:04:05+00:00",
"device_name": "chrome 121",
"ip": "192.168.50.1",
"exp": "2000-03-27T15:04:05+00:00"
}

## При выдаче нового refreshToken

- при выдаче нового refreshToken нужно обновлять запись в базе данных для устройства и продлить exp на 20 секунд (реализация в src\auth\adapters\jwt.service.ts функция createRefreshToken)
