## Задача

Нужно доработать auth-сервис так, чтобы у пользователя была привязка активной сессии к конкретному устройству.

## Текущее поведение

Пользователь логинится на одном устройстве и получает пару токенов: access и refresh.

Если он логинится с другого устройства, сервис выдает еще одну пару токенов.

Сейчас у одного пользователя может быть несколько активных сессий одновременно.

Эти сессии должны храниться отдельно и быть привязаны к разным устройствам.

## Что нужно реализовать

1. Создание сессии при логине
   При успешном логине нужно создавать запись сессии, привязанную к конкретному устройству.

2. Эндпоинт для устройств
   Добавить ручку:

GET /security/devices

Она должна возвращать список активных устройств пользователя.

3. Архитектурные ограничения
   Все GET-операции должны выполняться через qweryRepository.

qweryRepository нужно реализовать.

Прямой доступ к базе данных из контроллеров запрещен.

Доступ к БД должен быть только через слой repository.

Для CRUD-операций использовать слой service.

4. Работа с requestLogsRepository
   Нужно модифицировать requestLogsRepository.

Нужно также обновить используемую коллекцию в MongoDB.

## Требования к данным сессии

В базе можно использовать такую структуру:

user_id — id пользователя.

device_id — id устройства.

iat — время создания сессии.

device_name — название устройства, например chrome 121.

ip — IP-адрес клиента.

exp — время истечения сессии.

## Пример записи

json
{
"user_id": 1,
"device_id": 1,
"iat": "2000-02-27T15:04:05+00:00",
"device_name": "chrome 121",
"ip": "192.168.50.1",
"exp": "2000-03-27T15:04:05+00:00"
}

## Обновление refresh token

При выдаче нового refreshToken нужно:

обновлять запись в базе данных для соответствующего устройства;

продлевать exp на 20 секунд;

реализацию сделать в src/auth/adapters/jwt.service.ts, функция createRefreshToken.

## Логика user_id

user_id брать из JWT.

## Критерии готового решения

При логине создается сессия, связанная с конкретным устройством.

GET /security/devices возвращает список активных устройств текущего пользователя.

Все чтения идут через qweryRepository.

Все изменения данных идут через repository / service слой без прямого обращения к БД из контроллеров.

refreshToken обновляет запись устройства и exp на 20 секунд.

При реализации учитывай правила из src/prompts/common rules.md
